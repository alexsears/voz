🎤
Voice-First
Talk to Voz naturally. Say "tell my-api to add a health endpoint" and it dispatches the task to the right Claude instance.
🚀
Parallel Execution
Multiple Claude Code instances work on different projects simultaneously. Each in its own tmux pane with full context.
🧠
Persistent Memory
Every session automatically saves important decisions, patterns, and context to MEMORY.md. Nothing gets forgotten between sessions.
📄
Zero Config
Just talk to Voz. Say "I have a project called my-app at ~/code/my-app" and it sets up everything — tmux window, memory, CLAUDE.md.
🖥
Web Dashboard
Live terminal output from every project. Send commands, check status, start and stop sessions — all from one page.
🔒
Runs Locally
Everything runs on your machine. Your code never leaves your computer. Just tmux, Node.js, and Claude Code CLI.

Get Started in 30 Seconds

# Clone and install git clone https://github.com/alexsears/voz.git cd voz/app && npm install && cd .. # Start Voz ./start.sh # Then just talk: "Add project my-api at ~/code/my-api — it's a REST backend" "Tell my-api to add a health endpoint" "How's my-api doing?" "Status of all projects"
